## 5. リスク評価 (現実)

**そのままprompt化は破滅級**:

- Daily Loss $150 制限 = SL 3% × ノーション$1000 = $30 損失 → **5連敗で breach**
- 平均的に 31連敗が発生 → リアル運用ではほぼ確実に1ヶ月以内に breach
- 月別ばらつき大 → 「次の月も同じパフォーマンス」 は楽観的すぎる

**安全化必須3点**:

1. **銘柄限定**: BTC/HYPE/ZEC/WLD/SOL のみ (ETH/LINK/NEAR 除外)
2. **連敗ストッパー**: 当日 -$80 でその日全停止 (既存) + **3連敗で 6時間停止** 新規追加
3. **サイズ縮小**: ノーション $1000 → **$500** (SL hit損失 $15、 5連敗で-$75、 残バッファ $75)

これでも完璧ではない (31連敗が来たら $15×31 = $465 で残高 $4,535 で breach は防げる、 ただし trade量1/2に)。

## 6. 推奨運用パス

### Phase 1: paper-paper 観察 (推奨期間 1週間)

新 prompt 配信、 ただし **entry は markdown 出力のみ、 実発注しない** モード追加。 1週間後の **実 EV** と backtest 予測値 (+$2.90/trade) の乖離を見る。

期待結果:
- 乖離小 → 実弾化
- 乖離大 (EV-) → bot 完全停止、 戦略再設計

### Phase 2: 縮小実弾 (paper結果OKなら)

- ノーション $500
- 銘柄 BTC/HYPE のみ (最高勝率2銘柄に絞ってさらに保守)
- 連敗ストッパー有効
- 残高 $4,950 になったら強制停止

### Phase 3: 段階的拡大 (Phase 2 で2週間 EV+ なら)

- ノーション $750 → $1000
- 銘柄を ZEC/WLD/SOL 追加
- ただし $25k upgrade は3ヶ月EV+ 蓄積後

## 8. 結論

| 質問 | 回答 |
|---|---|
| 現prompt のままで運用してOKか | **絶対NO** (EV -$1.20/trade、 確実に手数料負け) |
| 探索でエッジ見つかったか | **Yes** (volume_spike_momentum、 OOS EV +$2.90) |
| そのまま実弾化していいか | **NO** (連敗31回、 銘柄ばらつき、 月別不安定) |
| どうすべきか | **Phase 1: paper観察1週間 → Phase 2: 縮小実弾** |

次のステップ案:
- (a) 現 prompt を破棄、 新prompt (volume_spike_momentum 専用、 paper-paperモード) で書き直し
- (b) bot 完全停止して人間の戦略再考に切り替え
- (c) backtest 期間延長 (5m足で7日、 1h足で208日 → 4h足で800日) で再探索

おすすめは (a)。 backtest で EV+ が確認できた唯一のロジックなので、 これを慎重に paper運用するのが最も学習効率高い。
